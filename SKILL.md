# FMHY.net Single-Page Search

## Overview

FMHY (Free Media Heck Yeah) maintains a massive wiki of free media, tools, and resources at https://fmhy.net. They expose a single-page API that dumps the entire site as one markdown file, making it perfect for command-line searching.

## When to Use

- Looking for a specific free tool, streaming site, software alternative, or resource
- "What's a good ...?" questions where FMHY likely catalogs it
- Need to bypass paywalls, find adblockers, get VPN recs, locate ebook sources, etc.
- Want to cross-check if a site/tool is listed and what the community says about it

**Don't use for:**
- Real-time data (API is a static snapshot)
- Content that isn't in the FMHY wiki (niche stuff they don't cover)

## The API

**Endpoint:** `https://api.fmhy.net/single-page`

Returns the entire FMHY wiki as a single markdown document (~1.8 MB, ~21k lines). It's autogenerated вЂ” do not edit it manually.

## Usage Pattern

FMHY updates frequently. Always grab a fresh copy, then do all your searches against that local file before cleaning up.

```bash
# 1. Download fresh to a temp file
TMPFILE=$(mktemp)
curl -s "https://api.fmhy.net/single-page" > "$TMPFILE"

# 2. Search as many times as you need during this run
rg -i "vpn" "$TMPFILE"
sed -n '/# в–є Gaming/,/# в–є /p' "$TMPFILE" | rg -i "emulator"

# 3. Clean up when done
rm -f "$TMPFILE"
```

Requires a Unix-like environment (Linux, macOS, WSL, Git Bash) with `curl`, `grep`, and optionally `rg` (ripgrep) installed.

## Quick Search Patterns

### Basic grep

```bash
grep -i "vpn" "$TMPFILE"
grep -i -C 3 "audiobook" "$TMPFILE"
grep -A 50 "в–є Audio" "$TMPFILE" | grep -i "spotify"
```

### With ripgrep (faster, prettier)

```bash
rg -i "self-hosted.*stream" "$TMPFILE"
rg -i "download.*manager" "$TMPFILE" --json | jq -r '.data.lines.text'
```

### Browse mode (when you don't know the keyword)

```bash
# 1. List all top-level categories
grep "^# в–є" "$TMPFILE"

# 2. Pick one and see everything inside it
sed -n '/# в–є Gaming/,/# в–є /p' "$TMPFILE" | head -n 80

# 3. Search within that section
sed -n '/# в–є Gaming/,/# в–є /p' "$TMPFILE" | rg -i "emulator"
```

### Section-first navigation

FMHY uses a consistent markdown structure:

- `# в–є Major Category` вЂ” top-level buckets (Adblocking, Audio, Video, Reading, Gaming, etc.)
- `## в–· Subcategory` вЂ” specific topics within the bucket
- `***` вЂ” horizontal rules separate sections
- `в–і` вЂ” sometimes marks beginner guide / getting-started sections

```bash
# List all top-level categories
grep "^# в–є" "$TMPFILE"

# Drill into a subcategory
sed -n '/## в–· Linux/,/## в–· /p' "$TMPFILE" | head -n 50
```

## One-Shot Recipes

### Browse mode: "I need a TTS tool but don't know names"
```bash

```

## Common Pitfalls

1. **The file is huge (~1.8 MB).** Run the `curl` once per session, do all searches against `$TMPFILE`, then `rm` it. Don't re-download for every grep.

2. **Grep without `-i` misses a lot.** Markdown links and labels have inconsistent casing.

3. **Cross-links (`в†ЄпёЏ`) lead to other sections.** A hit might just be a pointer, not a listing. Follow the cross-link or grep the target section.

4. **Starred items (`в­ђ`) are curated picks, not exhaustive lists.** If the starred hit isn't what you need, keep scrolling вЂ” the rest of the bullet list usually has more options.

5. **Rate limits are undocumented.** Don't hammer the endpoint in a tight loop; that's why we write to `$TMPFILE` within a run.

6. **Some sections are just links to Reddit wiki pages.** The single-page API includes these, but if a section looks thin, the real content may live on Reddit.
